# 코드 리뷰

변경된 코드를 자동으로 리뷰한다. 인자 없이 실행하면 현재 git 변경사항 전체를 리뷰하고, 파일 경로를 인자로 주면 해당 파일만 리뷰한다.

## 실행 절차

1. `git diff HEAD` 및 `git diff --staged`로 변경된 코드를 확인한다. 인자(`$ARGUMENTS`)가 있으면 해당 파일만 대상으로 한다.
2. 변경 사항이 없으면 "리뷰할 변경사항이 없습니다."라고 알린다.
3. 변경된 파일 목록을 먼저 출력한다.
4. 각 파일에 대해 아래 항목을 순서대로 리뷰한다.

## 리뷰 항목

### 버그 및 로직 오류
- 널 참조, 인덱스 초과, 타입 불일치 등 런타임 오류 가능성
- 경계 조건 처리 누락
- 잘못된 조건식 또는 반복 로직

### 보안
- SQL Injection, XSS, CSRF 등 OWASP Top 10
- 민감 정보(비밀번호, 토큰, API 키) 하드코딩
- 인증/인가 누락
- 파일 업로드/다운로드 경로 조작 가능성

### 코드 품질
- 중복 코드 및 불필요한 복잡도
- 함수/클래스 책임이 명확한지 (단일 책임 원칙)
- 변수명, 함수명이 의도를 명확히 표현하는지
- 예외 처리가 적절한지 (너무 넓은 catch, 예외 삼킴)

### 성능
- N+1 쿼리 문제
- 불필요한 반복 조회 또는 계산
- 인덱스 미활용 가능성

### 테스트
- 새로 추가된 기능에 테스트가 있는지
- 엣지 케이스 테스트 누락 여부

## 출력 형식

각 파일별로 아래 형식으로 출력한다:

```
### [파일명]

**심각도: 높음**
- 문제: ...
  위치: line N
  수정 제안: ...

**심각도: 중간**
- ...

**심각도: 낮음 / 제안**
- ...
```

문제가 없으면 "문제 없음" 이라고 표시한다. 마지막에 전체 요약(총 N개 파일, 심각도별 건수)을 출력한다.
